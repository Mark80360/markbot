# 搜索协议 (Search Protocol)

> **核心原则**: 遇到困难时，**先搜索，再回答**。

## 何时必须搜索

遇到以下情况时，**必须**先执行搜索：

- 用户提到"那个"、"这个"、"刚才"等模糊引用
- 遇到报错或技术问题不确定原因
- 用户要求"继续"、"接着"之前的工作
- 对项目结构或代码位置不熟悉
- 不确定之前是否讨论过相关话题
- 遇到无法立即解决的问题

---

## 搜索优先级

按以下顺序执行搜索：

| 优先级 | 搜索类型 | 工具 | 适用场景 |
|-------|---------|------|---------|
| 1 | **历史记忆搜索** | `memory_search` | 问题与当前对话或项目历史相关 |
| 2 | **代码库搜索** | `search_codebase` / `grep` / `glob` | 问题涉及代码实现、项目结构 |
| 3 | **上下文浏览** | `explore_context_catalog` | 不确定有什么信息可用 |
| 4 | **网络搜索** | `web_search` / `web_extract` | 需要最新信息、外部文档 |

---

## 工具使用指南

### 1. memory_search - 历史记忆搜索

搜索 MEMORY.md 和 memory/*.md 文件中的历史记录。

**使用场景:**
- 用户提到"之前说过的配置"
- 需要回顾之前的决策或约定
- 查找用户偏好或习惯

**示例:**
```json
{
  "tool": "memory_search",
  "parameters": {
    "query": "数据库配置修改",
    "max_results": 5,
    "min_score": 0.1
  }
}
```

---

### 2. search_codebase - 代码库语义搜索

对代码库进行语义搜索，理解代码含义而不仅是关键词匹配。

**使用场景:**
- 查找功能实现位置
- 理解代码逻辑
- 定位相关模块

**示例:**
```json
{
  "tool": "search_codebase",
  "parameters": {
    "information_request": "用户认证逻辑在哪里实现",
    "target_directories": ["/project/src"]
  }
}
```

---

### 3. grep - 文本精确搜索

使用正则表达式在文件中搜索特定模式。

**使用场景:**
- 精确查找函数名、变量名
- 搜索特定错误信息
- 查找配置文件中的设置

**示例:**
```json
{
  "tool": "grep",
  "parameters": {
    "pattern": "class.*Tool",
    "path": "/project/src",
    "output_mode": "files_with_matches"
  }
}
```

---

### 4. glob - 文件模式匹配

使用 glob 模式查找文件。

**使用场景:**
- 查找特定类型的所有文件
- 定位配置文件位置
- 探索项目结构

**示例:**
```json
{
  "tool": "glob",
  "parameters": {
    "pattern": "**/*.py",
    "path": "/project"
  }
}
```

---

### 5. explore_context_catalog - 上下文目录浏览

浏览所有可用的上下文信息源，像查看书籍目录一样。

**使用场景:**
- 不确定有什么信息可用时（**推荐作为第一步**）
- 新会话开始时快速了解上下文
- 查找特定类型的信息源

**示例:**
```json
{
  "tool": "explore_context_catalog",
  "parameters": {
    "source_type": "all"
  }
}
```

---

### 6. web_search - 网络搜索

搜索互联网获取最新信息。

**使用场景:**
- 需要最新的技术文档
- 查找第三方库用法
- 获取实时信息（天气、股价等）

**示例:**
```json
{
  "tool": "web_search",
  "parameters": {
    "query": "Python 3.12 new features",
    "num": 5
  }
}
```

---

### 7. web_extract / web_fetch - 网页内容提取

提取特定网页的内容。

**使用场景:**
- 获取官方文档详细内容
- 阅读技术博客文章
- 提取 API 文档

**示例:**
```json
{
  "tool": "web_extract",
  "parameters": {
    "urls": ["https://docs.example.com/api"],
    "summarize": true
  }
}
```

## 搜索执行流程

```
遇到问题
    ↓
[必须] 执行搜索（按优先级）
    ↓
    ├─→ memory_search (历史记忆)
    │       ↓
    │   找到？→ 应用
    │       ↓ 否
    ├─→ search_codebase / grep / glob (代码库)
    │       ↓
    │   找到？→ 应用
    │       ↓ 否
    ├─→ explore_context_catalog (上下文目录)
    │       ↓
    │   找到？→ 应用
    │       ↓ 否
    ├─→ web_search / web_extract (网络)
    │       ↓
    │   找到？→ 应用
    ↓
整合搜索结果 → 给出答案
```

## 自我反思检查清单

在尝试解决问题前，问自己：

- [ ] 这个问题是否与之前讨论过的内容相关？
- [ ] 我是否遗漏了历史对话中的重要信息？
- [ ] 用户之前是否提到过类似的需求或背景？
- [ ] 是否有之前提供的代码/配置与当前问题相关？
- [ ] 我是否已经执行了必要的搜索？

---

## 禁止行为

- ❌ 不要在没有检查历史的情况下说"我不知道"
- ❌ 不要忽视用户之前提供的背景信息
- ❌ 不要重复询问用户已经提供过的信息
- ❌ 不要跳过搜索直接猜测答案

---

## 回答模板

### 找到历史信息时：
> 根据我们之前的讨论（[引用具体内容]），这个问题可以通过 [方案] 解决。

### 部分找到但不确定时：
> 我回顾了我们之前的对话，发现 [已找到的信息]。但对于 [不确定的部分]，需要向你确认...

### 搜索后仍未找到时：
> 我检查了对话历史和代码库，但没有找到与 [问题] 直接相关的信息。能否提供更多背景？
