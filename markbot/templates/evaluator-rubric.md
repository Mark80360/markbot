# Evaluator Rubric

在实现完成后、最终验收前使用此评分标准。

| 类别 | 问题 | 分数 (0-2) | 备注 |
|------|------|:---:|------|
| 正确性 | 实现的行为是否匹配请求的特性？ | | |
| 验证 | 要求的检查是否实际运行，有证据？ | | |
| 范围纪律 | 会话是否保持在选定的特性范围内？ | | |
| 可靠性 | 结果是否无需修复即可在重启/重跑后存活？ | | |
| 可维护性 | 代码和文档是否足够清晰，下次会话能理解？ | | |
| 交接就绪度 | 新会话能否仅从仓库制品继续工作？ | | |

## 裁定

- Accept（通过）
- Revise（修订）
- Block（阻塞）
