# Quality Document

每个产品域和架构层的质量快照。Agent 和人类均可使用此文档快速了解代码库的强项和需要改进的地方。

**更新频率：** 每次重要会话后，或开始新阶段工作前。

**评分标准：**

- **A**: 所有验证通过，架构清晰，Agent 可理解，测试稳定
- **B**: 验证通过，基本清晰，可理解性或测试覆盖有小缺口
- **C**: 部分工作，已知缺口，部分代码区域 Agent 难以理解
- **D**: 不工作，或重大结构性问题
